# Module 04 — Building & Iterating

In this module you will connect the app's hard-coded mock data to your real Dataverse tables and iterate on the app with Kilo Code.

## Prompts

### 1. Plan the data connection

Switch Kilo Code into **Plan mode** and submit the following prompt:

```
Write up a plan for replacing the mock data in the app with real connections to the Dataverse tables
```

Review the plan carefully — check that every page and component that uses mock data is accounted for. When you are satisfied, approve the plan and let Kilo Code start a new session to implement it.

### 2. Iterate

Once the real data is wired up, use this session to iterate on the app. Some ideas:

- Add filtering or sorting to the asset list
- Improve the form validation
- Polish the UI to better match the design contract
- Add a confirmation dialog before deleting a record

There are no fixed prompts for this section — use what you have learned across the day to drive Kilo Code toward the outcome you want.

## Tips for effective iteration

- Keep prompts focused on one change at a time
- Use Plan mode when a change touches multiple files
- After each change, run the app and verify it works before asking for the next change
